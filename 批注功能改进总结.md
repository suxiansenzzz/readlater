# 批注功能改进总结

## 问题分析

### 原有问题
1. **批注按钮定位问题**
   - 按钮使用`position:fixed`但定位计算使用了`window.scrollY`
   - 导致按钮出现在错误位置
   - 没有边界检查，可能超出屏幕

2. **批注加载功能问题**
   - 使用简单的`innerHTML`替换
   - 无法处理HTML标签中的文本
   - 高亮文本被HTML标签分割时失效

3. **批注弹窗位置问题**
   - 弹窗固定出现在屏幕中央
   - 用户希望弹窗跟随选中文本

## 改进方案

### 1. 批注按钮定位改进
```javascript
// 修改前
btn.style.top = (rect.bottom + window.scrollY + 5) + 'px';
btn.style.left = Math.min(rect.right - 40, window.innerWidth - 60) + 'px';

// 修改后
let top = rect.bottom + 5;  // 不需要加scrollY
let left = rect.right - 40;

// 边界检查
if (top + 40 > window.innerHeight) {
    top = rect.top - 40; // 放到选中文本上方
}
if (left + 60 > window.innerWidth) {
    left = window.innerWidth - 60;
}
if (left < 10) {
    left = 10;
}
```

### 2. 批注高亮加载改进
**原方案问题：**
- 使用`innerHTML`替换
- 无法处理HTML标签
- 正则表达式转义复杂

**新方案：使用TreeWalker**
```javascript
// 使用TreeWalker遍历文本节点
const walker = document.createTreeWalker(
    articleBody,
    NodeFilter.SHOW_TEXT,
    null,
    false
);

let node;
const nodesToReplace = [];

// 查找包含高亮文本的文本节点
while (node = walker.nextNode()) {
    const text = node.textContent;
    if (text.includes(highlightText)) {
        nodesToReplace.push({
            node: node,
            text: text,
            highlightText: highlightText
        });
    }
}

// 替换找到的文本节点
nodesToReplace.forEach(item => {
    const { node, text, highlightText } = item;
    const index = text.indexOf(highlightText);
    
    if (index !== -1) {
        // 创建新的节点
        const beforeText = text.substring(0, index);
        const afterText = text.substring(index + highlightText.length);
        
        // 创建高亮span
        const highlightSpan = document.createElement('span');
        highlightSpan.className = 'highlight-text';
        highlightSpan.setAttribute('data-color', colorClass);
        highlightSpan.setAttribute('data-annotation-id', ann.id);
        highlightSpan.setAttribute('onclick', `showAnnotationDetail(${ann.id})`);
        highlightSpan.setAttribute('title', ann.note || '点击查看批注');
        highlightSpan.textContent = highlightText;
        
        // 创建文档片段
        const fragment = document.createDocumentFragment();
        
        if (beforeText) {
            fragment.appendChild(document.createTextNode(beforeText));
        }
        fragment.appendChild(highlightSpan);
        if (afterText) {
            fragment.appendChild(document.createTextNode(afterText));
        }
        
        // 替换原节点
        node.parentNode.replaceChild(fragment, node);
    }
});
```

### 3. 批注弹窗定位改进
```javascript
// 保存选中文本的位置信息
selectedTextData = {
    text: text,
    start: range.startOffset,
    end: range.endOffset,
    rect: {  // 保存选中文本的位置信息
        top: rect.top,
        bottom: rect.bottom,
        left: rect.left,
        right: rect.right
    }
};

// 定位弹窗到选中文本附近
if (selectedTextData.rect) {
    const rect = selectedTextData.rect;
    let top = rect.bottom + 10;
    let left = rect.left;
    
    // 边界检查
    if (top + 200 > window.innerHeight) {
        top = rect.top - 210; // 放到选中文本上方
    }
    if (left + 320 > window.innerWidth) {
        left = window.innerWidth - 330;
    }
    if (left < 10) {
        left = 10;
    }
    
    popup.style.top = top + 'px';
    popup.style.left = left + 'px';
}
```

## 改进效果

### 1. 用户体验改进
- 批注按钮出现在选中文本附近，符合用户预期
- 批注弹窗跟随选中文本，不遮挡内容
- 边界检查确保UI元素不超出屏幕

### 2. 功能稳定性改进
- TreeWalker方法正确处理HTML标签
- 避免了正则表达式转义问题
- 不会破坏原有HTML结构

### 3. 代码质量改进
- 使用DOM API而不是字符串操作
- 更好的错误处理
- 更清晰的代码结构
